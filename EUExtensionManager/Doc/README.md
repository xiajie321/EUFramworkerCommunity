# EU 拓展管理器

## 概述

EU 拓展管理器是统一管理 EUFramework 框架所有拓展插件的工具。它提供了可视化的界面来查看、安装、更新和卸载拓展插件，并支持从社区仓库获取最新资源。

## 功能特性

### 已安装管理
- **本地列表**：查看所有本地安装的拓展列表
- **版本状态**：实时显示版本信息和更新提示
- **快捷操作**：支持查看文档、定位文件夹、卸载等操作
- **自动扫描**：自动扫描 Assets、Packages 及核心目录下的拓展

### 社区仓库
- **远程浏览**：浏览远程社区仓库中的所有可用拓展
- **实时同步**：基于 GitHub API 获取实时列表
- **状态标记**：清晰标识已安装、未安装、可更新状态
- **一键安装**：支持 ZIP 下载与解压，自动处理依赖
- **智能缓存**：本地与内存双重缓存机制，提升加载速度

### 版本与依赖管理
- **版本检测**：自动检测可更新的拓展，支持一键更新
- **依赖解析**：自动解析并显示拓展依赖关系
- **递归安装**：支持递归安装依赖链，确保环境完整

### 核心框架支持
- **核心识别**：自动识别核心框架组件 (Category 为 "框架" 或 "Core")
- **独立路径**：支持将核心组件安装到独立的核心路径

### 设置与迁移
- **路径配置**：支持自定义插件安装路径和核心库路径
- **自动迁移**：修改路径时支持自动迁移现有拓展文件
- **安全安装**：同步文件内容而非简单覆盖，自动清理冗余文件

## 使用方法

### 打开管理器

菜单栏：`EUFramework` -> `拓展管理器`

### 界面说明

#### 左侧边栏
- **已安装**：查看本地已安装的拓展
- **社区仓库**：浏览远程可用的拓展
- **设置**：配置仓库地址和安装路径

#### 主内容区
- **顶部**：标题、搜索框、筛选器 (分类/排序)、刷新按钮
- **左侧列表**：拓展列表，显示图标、名称、版本、描述
- **右侧详情**：选中拓展的详细信息、依赖列表和操作按钮

### 操作按钮说明

| 按钮 | 说明 |
| --- | --- |
| 安装 | 从社区仓库下载并安装拓展 |
| 更新 | 更新到最新版本 (有新版时显示) |
| 文档 | 打开拓展的文档文件 (优先打开 README.md) |
| 定位 | 在文件管理器中打开拓展目录 |
| 卸载 | 删除拓展 (需确认，管理器自身不可卸载) |

## 设置选项

### 社区仓库地址
默认：`https://github.com/xiajie321/EUFramworkerCommunity`
用于获取拓展列表的 GitHub 仓库地址。

### 插件安装路径
默认：`Assets/EUFramework/Extension`
普通拓展插件的默认安装位置。

### 核心安装路径
默认：`Assets/EUFramework/Core`
核心框架组件的默认安装位置。

## 创建自己的拓展

每个拓展需要在根目录包含 `extension.json` 文件。详细格式请参考 [API 文档](API.md)。

### 推荐目录结构

```
YourExtension/
├── extension.json      # 必需：拓展描述文件
├── Script/             # C# 脚本
├── ConfigPanel/        # UI 样式文件
├── Doc/                # 文档目录
│   └── README.md       # 使用说明
└── Example/            # 示例
    ├── Scenes/
    └── Script/
```

## 常见问题

**Q: 为什么刷新列表很慢？**
A: 首次加载需要请求 GitHub API，受网络环境影响。后续加载会使用本地缓存 (5分钟有效)，速度会很快。

**Q: 如何手动安装拓展？**
A: 将拓展文件夹复制到项目的插件安装路径下，点击管理器刷新按钮即可。

**Q: 依赖安装失败怎么办？**
A: 检查网络连接，或手动下载依赖包并放置到指定目录。管理器会自动识别已手动安装的依赖。
